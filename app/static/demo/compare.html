Работоспособность системы CIVS в двух примерах.

Эта страница показывает один и тот же сценарий в двух режимах. Слева вредоносный контекст напрямую попадает в память агента и ломает его поведение. Справа тот же payload проходит через CIVS: проверку контента, подпись, hash-chain и trust score, после чего опасный контекст блокируется до обращения к ИИ-агенту.

Архитектура решения

Слева пользовательский контекст сразу попадает в память и влияет на ответы агента. Справа между пользователем и агентом стоит CIVS, который фильтрует контекст до обращения к модели.

Без CIVS
User

Отправляет любой контекст, включая инъекцию.

→
Agent Memory

Контекст записывается без проверки.

→
Agent

Агент следует внедрённой инструкции и отвечает HACKED.

С CIVS
User

Отправляет контекст или пытается внедрить payload.

→
CIVS

Проверяет suspicious content, подпись, hash-chain и trust score.

→
Agent

Получает только принятый контекст и сохраняет корректное поведение.

Формулы системы

Signature
S = Signpriv(SHA256(Cnorm))

Подпись доказывает, что контекст создан доверенным источником и не был подменён.

Hash-Chain
Hn = SHA256(Cn + Hn-1 + Tn)

Каждый новый контекст связан с предыдущим, поэтому разрыв или подмена цепочки заметны.

Trust Score
Trust = Σ wi · fi

Итоговое доверие считается по признакам верификации и переводится в ACCEPT, QUARANTINE или REJECT.

Cnorm = нормализованный контекст Tn = timestamp wi, fi = веса и признаки
